# Pipeline: highlights

**Makes:** a 9:16 (1080×1920, ~25s) auto-highlights reel: source clips repurposed inside a branded "analysis UI" — windowed player, draw-on detection circles with confidence tags, a scanned-match timeline with moment markers, intro scan + end card. No narration; whoosh/blip SFX carry the rhythm.
**When:** "highlights", "repurpose these clips", "clips reel" — you have 2–4 short source clips and want them packaged as an AI-detected-moments reel.
**Master rules:** `references/VIDEO_PLAYBOOK.md` — determinism, QA, render loop apply. No VO/transcribe/cues steps: timing is authored directly in build.py.

## Worked example anatomy (`templates/highlights/`)

```
build.py          # THE source of truth — CLIPS schema, section timings, all markup + GSAP tweens
index.html        # GENERATED by build.py — never hand-edit
clip1-3.mp4       # the source highlight clips (swap these)
whoosh.mp3        # transition SFX — fires at every clip start and at the end card (track 30)
blip.mp3          # detection SFX — fires at every circle pop (track 31)
videodb-logo.png  # brand mark (topbar, intro, end card, badge)
CLAUDE.md / AGENTS.md · meta.json · hyperframes.json · package.json · template.json (workflow steps)
```

## The build.py schema

**Sections** (three phases on one timeline):
- `INTRO_S..INTRO_E` (0–2.6): logo + "AUTO-HIGHLIGHTS" title + match subtitle + an orange scan-line sweeping down.
- `ANALYSIS_S..ANALYSIS_E` (2.6–19.6): the clips play in sequence under persistent chrome — topbar (logo + pulsing rec-dot "AUTO-HIGHLIGHTS" pill), the scan timeline, and a "Made with …" badge.
- `END_S..END_E` (19.6–24.2): big logo, tagline, sub-line, pulsing CTA button.

**Brand block:** `DARK/PANEL/ORANGE/INK` inside `BRAND:START/END` — rewritten from `brand.json` on scaffold.

**Window rects:** `LAND` (16:9 window, l/t/w/h) and `VERT` (9:16 window) — each clip picks one via `rect=`, so mixed-orientation sources sit in correctly shaped bordered player windows.

**`CLIPS = [dict(...)]`** — one entry per highlight, fields:
| field | meaning |
|---|---|
| `src` | source file |
| `ds`, `dur` | start on the reel timeline, on-screen duration (clips must tile `ANALYSIS_S → ANALYSIS_E` contiguously) |
| `min` | media in-point (seconds into the source, via `data-media-start`) |
| `pos` | object-position framing inside the window |
| `rect` | `LAND` or `VERT` |
| `cx, cy, r` | detection-circle center + radius in canvas px (position it over the action) |
| `pop` | absolute time the detection fires (circle + tag pop, blip SFX) — pick the moment the action peaks |
| `tag`, `conf` | detection chip text ("GOAL DETECTED") + confidence string ("0.98") |
| `label`, `time` | big moment label under the window ("GOAL", "23'") |
| `mark` | match-minute → marker position on the scan timeline via `mark_x()` |

**Header/timeline constants:** `MATCH`, `COMP`, `TOTAL` (scanned footage duration string), `MATCH_MIN` (total match minutes for marker math), `TL_L/TL_R` (timeline x-range). The playhead tweens to each clip's marker at `ds+0.02`.

**Generated per clip:** the `<video>` (tracks 0–2, `muted playsinline`), detection circle (solid ring + rotating dashed "scanning" ring; tracks 3–5), tag chip (tracks 6–8), moment label (tracks 9–11) — each timed with `data-start`/`data-duration` derived from `ds/dur/pop`. **SFX:** `WHOOSH = [each ds] + [END_S]`, `BLIPS = [each pop]`.

**Tweens (all authored in build.py, one paused timeline):** intro title mask-reveal + scan sweep; chrome slide-ins at `ANALYSIS_S`; rec-dot pulse (finite `repeat:14`); per clip — fade+settle (scale 1.08→1.0 across `dur`), label slide-in at `ds+0.15`, circle pop `back.out(2)` at `pop-0.05` with 5-rep ring pulse and full-duration dash rotation, tag pop at `pop+0.05`; end card staggered entrances + 3-rep CTA pulse.

## Build workflow (clip replacement)

1. Scaffold: `python3 "$SKILL_DIR/scripts/new_project.py" highlights <slug>`; watch the example (`npm install && npx hyperframes preview`).
2. Replace `clip1-3.mp4` with your source clips (add/remove entries freely — everything downstream is generated from `CLIPS`).
3. For each clip, set `min` (scrub the source for the action), `dur`, `rect` + `pos` (orientation/framing), `cx/cy/r` (circle over the subject at pop time), `pop` (the action's peak), `tag/conf/label/time/mark`.
4. Update the story constants: `MATCH`, `COMP`, `TOTAL`, `MATCH_MIN`; retime `INTRO/ANALYSIS/END` so `ANALYSIS_E` = last clip's `ds+dur` and the end card gets ~4.5s.
5. Regenerate: `python3 build.py` → writes `index.html`. **Never hand-edit index.html** — it's clobbered on the next run.
6. Render loop: `npx hyperframes lint` → `npx hyperframes render -o out.mp4 --quality draft` → extract frames to verify circle placement over the actual action (the one thing lint can't check) → adjust `cx/cy/r/pop` in build.py → rerun build.py → re-render → `--quality high` when it lands.

## Format-specific gotchas

- `cx/cy` are canvas coordinates, not window-local — position circles against a rendered frame, not the source clip.
- `pop` is absolute reel time; it must fall inside `[ds, ds+dur]` or the circle's `data-duration` goes negative.
- Contiguous clip tiling: a gap between one clip's end and the next `ds` shows bare background under the chrome.
- Clips are `muted` by design — this format has no voice track; if you add crowd audio, put it on its own track and keep SFX tracks 30/31.
- The rec-dot pulse and ring pulses use finite `repeat:` counts sized to the section length — retime them if you stretch `ANALYSIS`.
- Vertical safe zones matter here too (it ships to Reels/Shorts): the example keeps the timeline at y1640 and badge at bottom:60 — if your platform overlay eats them, raise them into the center band.
