# Pipeline: split-ugc

**Makes:** a 9:16 (1080×1920, 30fps, ~40s) UGC-style creator video: a talking head opens full-screen, docks into the bottom half, and animated cards run in the top half, with big white captions riding the seam and a click SFX on every card change.
**When:** "split screen", "ugc", "talking head + cards", "creator video" — you have (or will record) a talking-head clip and want it packaged into the split format.
**Master rules:** `references/VIDEO_PLAYBOOK.md` — determinism, QA, render loop apply. There is no TTS/cues step here: the talking-head video IS the voice track; everything times to its transcript.

## Worked example anatomy (`templates/split-ugc/`)

```
build.py          # THE source of truth — CARDS + CAPTIONS schemas, layout constants, all markup/tween generators
index.html        # GENERATED by build.py — never hand-edit
Video1.mp4        # the talking-head source (video full-frame on track 0; same file as <audio> on track 1)
transcript.json   # word-level whisper transcript of Video1.mp4 — the timing source for CARDS and CAPTIONS
clip1-3.mp4       # b-roll clips for the montage cards (kind="clip")
*.jpg / *.png     # image-card stills, product-scene posters/thumbs, logo
click-sfx.mp3     # fires at every card start (track 4)
CLAUDE.md / AGENTS.md · meta.json · hyperframes.json · package.json · template.json (workflow steps)
```

## The format (constants at the top of build.py)

- `DURATION = 40.2` — total length = the talking-head clip's length.
- `DOCK = 3.6` — the avatar plays full-screen (slow scale 1.0→1.10) until DOCK, then the video-wrap slides to `VIDEO_TOP` over 0.55s and the top zone takes over.
- `SEAM = 812` — the top card zone is `0 → SEAM`px; `VIDEO_TOP = 760` — where the video wrap lands; `CAP_TOP = 828` — caption zone y; `VIDEO_OBJ_POS` — object-position keeping the face framed after the dock. A `seam-fade` gradient blends the boundary.
- `BRAND:START/END` block: `CARD/GRID/INK/CORAL/RED/DARK` theme tokens (rewritten from `brand.json` on scaffold) + a secondary literal product-brand color that is NOT a theme token.
- Track map: talking-head video `0`, its audio `1`, cards alternate `2`/`5`, captions alternate `3`/`6`, click SFX `4`, montage clips `10/11/12` (+3 for their label tags).

## CARDS schema (`CARDS = [dict(...)]`, one per top-zone beat)

Every card: `s` (start), `e` (end), `kind`. Cards get an entrance tween from a rotating `EASES` list and a click SFX at `s`. Kinds and their props:

| kind | props | renders |
|---|---|---|
| `image` | `img`, `pos` (object-position) | full-bleed still with a slow Ken Burns (scale→1.06) |
| `statement` | `top` (bold line), `acc` (italic serif accent), `color` | two-line kinetic type on the grid card |
| `word` | `acc`, `color` | one giant italic serif word (208px) popping in |
| `price` | `label`, `big`, `color` | label + huge stat/price line (228px, tabular-nums) |
| `clip` | `src`, `min` (media in-point s), `label`, `trk` (track index) | full-top-zone muted video + orange corner tag — emitted top-level, NOT nested, so the framework can play it |
| `logo` | `img` | logo on the dark grid card with a glow drop-shadow |
| `paste` | — | BESPOKE product scene 1 (see below) |
| `results` | — | BESPOKE product scene 2 (see below) |
| `cta` | `top`, `acc`, `sub`, `color` | closing "comment X" card |

Cards must tile the top zone contiguously from `DOCK` to `DURATION` (each `e` = next card's `s`).

## CAPTIONS schema

`CAPTIONS = [(start, "text"), ...]` — short 2–5-word phrases; each caption extends until the next one's start (last one to `DURATION`). Rendered as white 96px weight-900 text with a heavy shadow, centered in the seam zone (`CAP_TOP`), popping in with `back.out(2.6)`. Times come from `transcript.json` word starts — never guessed.

## The BESPOKE SCENES markers

Two `--- BESPOKE SCENES ---` comment blocks mark the product-specific demo (both markup and tweens): `PASTE_INNER` (a search bar → video result → "Link copied" → paste-into-product panel → Analyze button pulse) and `results_inner()` (product header + three poster result-cards staggering in). **To retarget the template at another product, replace exactly these two scenes' markup + their `card_tweens` branches** (`kind == "paste"` / `"results"`) with your product's UI, and swap the referenced images. Everything else is product-agnostic.

## Build workflow

1. Scaffold: `python3 "$SKILL_DIR/scripts/new_project.py" split-ugc <slug>`; watch the example (`npm install && npx hyperframes preview`).
2. Drop your talking-head video in as `Video1.mp4` (9:16, face in the upper part of frame — it gets cropped to the bottom half after DOCK; tune `VIDEO_OBJ_POS` if the face sits wrong).
3. Transcribe it: `npx hyperframes transcribe Video1.mp4` → `transcript.json`.
4. Re-time everything from the transcript: set `DURATION` to the clip length; pick `DOCK` at the end of the spoken hook (~3–4s); rewrite `CAPTIONS` as short phrases at real word-start times; rewrite `CARDS` so each card lands on the sentence it illustrates.
5. Replace the bespoke scenes (if it's a different product) and swap the images/clips the CARDS reference (set each `clip` card's `min` in-point to the action).
6. Regenerate: `python3 build.py` → writes `index.html`. **Never edit index.html by hand** — it's overwritten on every run; all edits go in build.py.
7. QA + render per the master playbook: `npx hyperframes lint && npx hyperframes render -o out.mp4 --quality draft`, eyeball frames, then `--quality high`.

## Format-specific gotchas

- `clip` cards need explicit non-colliding `trk` values (the example uses 10/11/12) because they're top-level siblings, not children of the top zone.
- The `<audio>` clip points at the same `Video1.mp4` (video element is `muted`; framework rule: video + separate audio element).
- Caption text is raw HTML — escape/curly quotes deliberately (the example uses typographic quotes in CTA text).
- Cards run edge-to-edge in time; a gap between `e` and the next `s` leaves the bare docked video with an empty top zone — only acceptable if intentional.
- Keep the click SFX volume at 0.5; it fires once per card, which is the format's rhythm — don't add extra SFX per tween.
- The whole video is one composition (`data-composition-id` matches the project slug) — no sub-compositions, no cues_spec/extract_cues.
